### Change 2 — Classifier rule priority swap (F31)

In Pass 0A's classifier (the function you added in the v0.5 cycle), swap the priority order of rules 3 and 4 per Row 4 v0.6 §4.2.5.

**Previous order (v0.5)**:
1. bullet item
2. table row
3. definition line
4. sentence in prose
5. multi-line block

**New order (v0.6)**:
1. bullet item
2. table row
3. **sentence in prose** ← moved up from position 4
4. **definition line** ← moved down from position 3
5. multi-line block

The rule logic itself stays the same:

- **Rule 3 (sentence in prose)**: If Source.source_text contains a sentence terminator (.!?) followed by whitespace or end-of-text → classify as "sentence in prose". This fires regardless of whether the Source begins with a label-colon prefix.
- **Rule 4 (definition line)**: Else if Source.source_text matches label-colon-value pattern where value is short and does NOT end in sentence terminator → classify as "definition line".

The priority swap means rule 4's "value does not end in sentence terminator" condition is now achieved by construction: any Source with a sentence terminator goes to rule 3 first; only Sources without sentence terminators get to rule 4. You may simplify rule 4's regex if you wish (the explicit "no sentence terminator in value" check becomes redundant because rule 3 caught those cases). Or keep the explicit check as defensive validation — either is acceptable.

**Expected behaviour change**:

The 6 misclassified policy clauses in TST-002 v2 (S719-S723, S725 — or v3-equivalents after the rerun) will now classify as "sentence in prose" instead of "definition line". This is the visible diagnostic that F31 is fixed.
