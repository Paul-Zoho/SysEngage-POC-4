## The implementation change

### F29 — Implement segmentation_context classifier in Pass 0A

Currently Pass 0A populates `Source.segmentation_context = "sentence in prose"` for every Source. Replace with classifier-derived value per Row 4 v0.5 §4.2.5.

**Classifier enumeration (5 values)**:

- `"sentence in prose"` — prose Source ending in sentence terminator
- `"bullet item"` — Source originating from a bulleted list per Pass 0 structural markers
- `"table row"` — Source originating from a table row per Pass 0 structural markers
- `"definition line"` — Source matching label-then-value pattern
- `"multi-line block"` — Source spanning multiple lines without sentence-end punctuation and not matching definition-line pattern; fallback for signature blocks, attribution blocks, structured non-prose

**Priority-ordered classification rules** (first matching rule determines the value):

1. If Pass 0 surfaced a bullet-list structural marker for this Source → `"bullet item"`
2. Else if Pass 0 surfaced a table-row structural marker for this Source → `"table row"`
3. Else if Source.source_text matches definition-line pattern (label followed by colon followed by value, e.g., `"Date: 22/4/2025"` or `"Title: Director"` or `"Owner : Damian Shacklock"`) → `"definition line"`
4. Else if Source.source_text contains a sentence terminator (`.!?`) followed by whitespace or end-of-text AND does not contain unmatched line breaks suggesting block structure → `"sentence in prose"`
5. Else (Source spans multiple lines without sentence punctuation and doesn't match definition-line pattern) → `"multi-line block"`

**Implementation guidance**:

- Rules are deterministic. Same Source content + same structural marker context produces identical segmentation_context across runs. No AI involvement.
- Bias toward conservative classification: when in doubt between "sentence in prose" and "multi-line block", prefer "multi-line block" if the Source clearly spans multiple lines without a terminator.
- Definition-line pattern detection: a regex like `^\s*[A-Za-z][A-Za-z\s]*\s*:\s*.+` works for simple cases (label is alphabetic with optional spaces; colon possibly preceded by whitespace; value follows). Tighten or loosen based on what cleanly handles the Novus signature block (`Date: 22/4/2025` and `Signed :` patterns).
- The classifier operates on the captured Source after Pass 0A boundary detection; it does NOT change Source boundaries.
